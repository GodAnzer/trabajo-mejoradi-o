Antes de programar

Tres conceptos que guían el proyecto

Este sistema no se evalúa solo por verse bonito. Se evalúa por validar, proteger datos y orientar al usuario cuando se equivoca.

01

Validación

Comprueba si un dato cumple reglas: obligatorio, formato, rango, longitud o selección.

02

Integridad de datos

Evita información incompleta, repetida, incoherente o imposible de utilizar.

03

UX

Facilita que el usuario entienda qué debe escribir, qué falló y cómo corregirlo.

Tu intervención

Retos mínimos que debes resolver

Reto 1: mejora una validación

Modifica `js/validators.js` para hacer más estricta o más clara una regla existente.

Reto 2: mejora la UX

Modifica `css/styles.css` para mejorar mensajes, contraste, botones o diseño móvil.

Reto 3: documenta pruebas

Completa `evidencias/MATRIZ_PRUEBAS.md` con datos válidos, inválidos y resultado observado.
