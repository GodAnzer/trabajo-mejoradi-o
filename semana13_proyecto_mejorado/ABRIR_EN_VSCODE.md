# Cómo abrir y trabajar este proyecto en Visual Studio Code

## 1. Descomprimir el archivo

Primero descomprime el ZIP. No trabajes directamente dentro del ZIP porque VS Code no podrá guardar correctamente algunos cambios.

## 2. Abrir carpeta completa

En VS Code debes abrir la carpeta completa, no solo un archivo suelto.

Ruta correcta:

```text
semana13_taller_practico_validacion_ux_profesional/
```

Ruta incorrecta:

```text
index.html
```

Abrir la carpeta completa permite ver HTML, CSS, JavaScript, guías, evidencias y pruebas en un mismo lugar.

## 3. Ejecutar el proyecto

### Opción A: abrir directamente

Busca `index.html` y ábrelo con el navegador.

### Opción B: usar Live Server

1. Instala la extensión **Live Server**.
2. Abre `index.html`.
3. Clic derecho.
4. Selecciona **Open with Live Server**.

Esta opción se parece más a un entorno de desarrollo real.

## 4. Usar la consola del navegador

Presiona `F12` o clic derecho > **Inspeccionar** > pestaña **Console**.

La consola sirve para verificar errores técnicos. Para la entrega, toma una captura donde se vea que no hay errores después de probar el formulario.

## 5. Archivos que vas a modificar

- `index.html`: estructura del formulario.
- `css/styles.css`: diseño visual y experiencia de usuario.
- `js/validators.js`: reglas de validación.
- `js/app.js`: eventos principales y flujo del sistema.

## 6. Archivos que vas a diligenciar

- `evidencias/MATRIZ_PRUEBAS.md`
- `evidencias/BITACORA.md`
- `evidencias/INFORME_ENTREGA.md`
