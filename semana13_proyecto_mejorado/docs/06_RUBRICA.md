# Rúbrica de evaluación

| Criterio | Superior | Alto | Básico | Bajo | Valor |
|---|---|---|---|---|---:|
| Ejecución del proyecto | Abre, ejecuta y explica la estructura completa. | Abre y ejecuta correctamente. | Ejecuta con ayuda. | No logra ejecutar. | 15% |
| Modificación de código | Modifica HTML, CSS y JS con sentido técnico. | Modifica al menos dos archivos correctamente. | Modifica parcialmente. | No modifica o rompe el proyecto. | 25% |
| Validación e integridad | Mejora reglas y controla errores/duplicados. | Aplica validaciones funcionales. | Reconoce validaciones pero con fallas. | No valida correctamente. | 25% |
| Pruebas | Prueba casos válidos, inválidos y duplicados con evidencias. | Presenta varias pruebas claras. | Presenta pocas pruebas. | No evidencia pruebas. | 20% |
| Documentación | Bitácora e informe explican cambios, errores y decisiones. | Documenta cambios principales. | Documenta de forma superficial. | No documenta. | 15% |

## Recomendación para autoevaluarte

Antes de entregar, revisa:

- ¿Mi proyecto abre sin errores?
- ¿Mis validaciones funcionan?
- ¿Puedo explicar qué cambié?
- ¿Tengo capturas suficientes?
- ¿La matriz de pruebas está completa?
